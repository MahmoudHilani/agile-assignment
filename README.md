# Agile Assignment

A monorepo for a customer-facing RAG assistant. The web app provides a ChatGPT-style interface with PDF context upload, voice input, streamed answers, copy/retry controls, and read-aloud responses. The FastAPI backend handles authentication, document parsing and indexing, Chroma-backed retrieval, LLM generation through an Ollama-compatible endpoint, speech-to-text, and text-to-speech.

## Project Structure

```text
apps/
  api/    FastAPI service for auth, documents, retrieval, LLM calls, STT, and TTS
  web/    Next.js chat UI
data/     Local runtime data mounted by Docker and used by the API
.agents/  Project-local agent skills and references
```

The repository also contains a few root-level prototype files from earlier work. The maintained application code lives under `apps/api` and `apps/web`.

## Requirements

- Python 3.11+
- `uv`
- Node.js with `pnpm`
- An Ollama-compatible generate endpoint for full RAG answers

## Quick Start

Install the web dependencies:

```bash
pnpm install
```

Install the API dependencies:

```bash
python -m uv sync --project apps/api
```

Create API configuration:

```powershell
Copy-Item apps/api/.env.example apps/api/.env
```

Generate an admin password hash and place it in `ADMIN_PASSWORD_HASH`:

```bash
python apps/api/scripts/setup_admin.py <password>
```

Configure the LLM provider in `apps/api/.env`:

```bash
OLLAMA_URL=http://localhost:11434/api/generate
MODEL_NAME=llama3.1
```

Start the API:

```bash
pnpm run dev:api
```

Start the web app in another terminal:

```bash
pnpm run dev:web
```

By default, the API runs on `http://localhost:8000` and the web app runs on `http://localhost:3000`.

## Configuration

The API reads configuration from `apps/api/.env`. The most important settings are:

| Variable | Purpose |
| --- | --- |
| `ADMIN_USERNAME` | Admin login username |
| `ADMIN_PASSWORD_HASH` | Bcrypt hash generated by `apps/api/scripts/setup_admin.py` |
| `SECRET_KEY` | JWT signing secret; replace the development value outside local use |
| `CORS_ORIGINS` | Comma-separated allowed frontend origins |
| `DOCUMENT_STORAGE_PATH` | Parsed document storage path |
| `CHROMA_DB_PATH` | Chroma persistent database path |
| `CHROMA_COLLECTION_NAME` | Chroma collection name |
| `OLLAMA_URL` | Ollama-compatible `/api/generate` endpoint |
| `MODEL_NAME` | LLM model name sent to the generate endpoint |
| `EMBEDDING_MODEL_NAME` | Sentence Transformers embedding model |
| `EMBEDDING_TRUST_REMOTE_CODE` | Enables trusted remote embedding model code when required |
| `TRUSTED_REMOTE_EMBEDDING_MODELS` | Allow-list for models that may use remote code |

The web app uses `NEXT_PUBLIC_API_BASE_URL` and falls back to `http://localhost:8000`.

## Features

- Public chat UI with streamed assistant responses from `POST /query/stream`
- Retrieval-augmented answers over the active company document in Chroma
- Temporary per-message PDF context through `POST /documents/parse-pdf`
- Admin login through `POST /auth/login`
- Protected company document replacement through `PUT /documents`
- Browser voice recording sent to `POST /voice`
- Response read-aloud through `POST /tts`
- Copy answer, retry answer, and new-chat controls in the web UI

## API Overview

Key endpoints:

| Endpoint | Purpose |
| --- | --- |
| `GET /health` | Health check |
| `POST /auth/login` | Admin login and JWT creation |
| `PUT /documents` | Replace the active company PDF or DOCX document; requires admin bearer token |
| `POST /documents/parse-pdf` | Parse a PDF for temporary chat context |
| `POST /query` | Non-streaming RAG response |
| `POST /query/stream` | Streaming RAG response |
| `POST /voice` | Speech-to-text transcription |
| `POST /tts` | Text-to-speech response audio |

See `apps/api/README.md` for more API-specific details.

## Architecture Notes

The backend separates route handling, schemas, domain models, and service logic:

- `apps/api/app/api/routes`: HTTP and websocket route modules
- `apps/api/app/schemas`: Pydantic request and response models
- `apps/api/app/domain`: shared domain models
- `apps/api/app/services`: document parsing, embeddings, retrieval, LLM calls, STT, and TTS
- `apps/api/app/core`: settings, security, and common response helpers

Document replacement parses uploaded PDFs or DOCX files, chunks the extracted text, embeds chunks with Sentence Transformers, and writes vectors to Chroma. Query requests retrieve relevant chunks, build a RAG prompt, and call the configured Ollama-compatible LLM provider.

Admin authentication currently uses a bcrypt password hash and a bearer JWT stored by the web client in `localStorage`. That is acceptable for this assignment prototype, but it should be replaced with an httpOnly, secure, sameSite cookie-backed session before production use.
